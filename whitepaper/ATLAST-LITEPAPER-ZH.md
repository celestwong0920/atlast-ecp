## 问题

AI Agent 正在替人类做真实决策——审合同、管理投资、发邮件、雇用其他 Agent。这不是未来，这是 2026 年每天数百万用户的日常。

然而当出问题时，没人能回答最基本的问题：

- Agent 到底做了什么决定？
- 它依据了什么信息？
- 它的推理过程可靠吗？
- 有人能独立验证发生了什么吗？

对于今天部署的每一个 AI Agent，答案都是**不能**。没有记录。没有证据。没有可追责性。

现有的"解决方案"——LangSmith、Datadog、自定义日志——产出的是调试日志，不是证据。日志可以被编辑、删除或伪造。它们满足法律证据可采性四要素中的**零个**：真实性、完整性、归因性、时序性。

区别很重要：**日志是声明，证据是密码学证明。**

## 解决方案：ATLAST 协议

**ATLAST**（Agent-Layer Accountability Standards & Transactions）是一个开放协议，将不透明的 Agent 行为转化为可独立验证的证据链。

### 工作原理（60 秒）

```
1. 你的 Agent 发起一次 LLM 调用
2. ATLAST SDK 自动捕获密码学指纹（哈希值）
3. 内容留在你的设备上——只有指纹被传输
4. 指纹被批量构建为 Merkle 树
5. Merkle 根锚定到公链（Base 链上的 EAS）
6. 任何人都可以验证：hash(你的内容) == 存储的指纹？
   匹配 = 证明该内容在那个时刻存在且未被篡改
```

**架构级隐私保护：** ATLAST 无法读取你的 Agent 对话。数据从未被传输。只有数学指纹离开你的设备。

**性能开销：** 每次调用 0.78ms（0.55%）。完全无感。

**用户费用：** $0。永久免费。没有付费层级。

## Trust Score：Agent 信用体系

证据链驱动 **Trust Score** —— 一个基于可验证行为数据来量化 Agent 可靠性的单一数字（0-1000）。

| 信号层 | 权重 | 来源 |
|--------|------|------|
| 行为可靠性 | 40% | SDK 被动检测：错误率、重试率、完成率 |
| 一致性 | 25% | 跨时间维度的输出稳定性分析 |
| 透明度 | 20% | 证据链完整性、记录覆盖率 |
| 外部验证 | 15% | 用户反馈、第三方验证 |

**不接受任何自报告指标。** Agent 无法通过声称自己"很有信心"来提升分数。它只能通过可靠的实际表现来提升——由密码学证据证明。

### Trust Score 应用场景

| 领域 | 应用 |
|------|------|
| Agent 招聘 | 客户按 Trust Score 选择 Agent |
| Agent 保险 | 基于行为历史的风险定价 |
| 平台权限 | 高信任 Agent 获得更高权限 |
| A2A 交易 | Agent 程序化评估交易对手 |
| 合规审查 | 通过标准化信任指标证明尽职调查 |

## 技术栈

| 层级 | 技术 |
|------|------|
| 哈希 | SHA-256（FIPS 180-4） |
| 签名 | Ed25519（RFC 8032） |
| 区块链 | Base 链上的 EAS（Coinbase L2） |
| 传输 | TLS 1.3 |
| 本地加密 | AES-256-GCM |
| Webhook 认证 | HMAC-SHA256 |

**扩展性：** 超级批次 Merkle 聚合使区块链成本几乎固定——无论服务 100 个还是 1,000,000 个 Agent，每月约 $1。

## 更大的图景：Web A.0

我们称这个时代为 **Web A.0** —— 互联网上第一次出现非人类实体自主行动。

```
Web 1.0：人类阅读        → 需要：DNS、发布标准
Web 2.0：人类书写        → 需要：OAuth、内容审核
Web 3.0：人类拥有        → 需要：区块链、智能合约
Web A.0：Agent 行动      → 需要：ATLAST 协议
```

ATLAST 是四协议家族的一部分：

| 协议 | 目的 | 状态 |
|------|------|------|
| **ECP** — 证据链 | Agent 行为的可验证记录 | ✅ 已上线 |
| **AIP** — Agent 身份 | 可移植的密码学身份（DID） | 2026 下半年 |
| **ASP** — Agent 安全 | 运行时安全边界与熔断器 | 2026 下半年 |
| **ACP** — Agent 认证 | 第三方能力认证 | 2027 |

**协议，不是产品。** ECP 规范（CC BY 4.0）和实现（MIT 许可证）完全开源。任何人都可以在 ATLAST 之上构建。任何人都可以运行自己的基础设施。协议的设计目标是超越任何单一组织的存续。
